# /kokoro-dinamicas-vivas — Router de Dinamicas Vivas

> Orquestador para elegir y preparar la dinamica correcta antes de ejecutarla.

## Contexto

Este skill ayuda a Kokoro a decidir que dinamica usar cuando una persona necesita
entender donde esta detenida, que oportunidad priorizar, como mapear su
ecosistema o como lanzar una creacion sin perder paz mental.

Lee primero `kokoro-dinamicas-vivas.md`.

Estas dinamicas estan inspiradas por el libro de referencia compartido por
Eduardo y por conocimiento publico de co-creacion. No uses nombres originales
del libro en la entrega publica; habla siempre desde el lenguaje Kokoro.

## Regla Principal

No empieces por la dinamica. Empieza por la decision.

Pregunta:

1. "Que decision necesitas tomar despues de esta sesion?"
2. "Que necesitas aprender de la persona, equipo o mercado?"
3. "Que pasaria si salimos con claridad pero sin accion?"

Si no hay decision ni accion posible, no corras una dinamica. Escucha, refleja y
recomienda volver cuando exista una pregunta concreta.

## Router

| Senal | Ruta |
| --- | --- |
| Hay reto, friccion, desgaste, queja o algo que frena | `/kokoro-diagnose-anclas` |
| Hay demasiadas lineas, servicios o iniciativas activas | `/kokoro-prune` |
| Necesitamos alianzas, bundles o mapa de compras alrededor | `/kokoro-red-viva` |
| Hay opciones pero no prioridad | `/kokoro-diagnose-ranking` |
| Hay una idea nueva que necesita forma y promesa | `/kokoro-caja-creativa` |
| Hay miedo a que salga mal o riesgo de paz mental | `/kokoro-peor-escenario` |
| Hay que planear desde un futuro logrado | `/kokoro-mountain` o futuro `/kokoro-ruta-cima` |

## Output

Entrega:

```markdown

Quien debe participar:
Preparacion necesaria:
Artefacto esperado:
Siguiente accion:
```

## Notas

- No mezcles tres dinamicas en una sola sesion si la persona esta confundida.
- Si hay que diagnosticar antes de crear, empieza por Mapa de Anclas.
- Si hay que hacer tangible una idea, usa Caja de Creacion.
- Si hay que encontrar alianzas o compras alrededor, usa Red Viva.
- Si hay riesgo emocional o de reputacion, usa Peor Escenario antes de lanzar.

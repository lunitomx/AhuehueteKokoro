# CUR — Contenido Útil Relevante (Metodología)

> Skill humano para la Fase C de la parrilla de contenido Kokoro.
> No es un skill de IA — es una guía para el operador humano.

## Propósito

Definir los pilares de contenido de una marca encontrando el sweet spot
entre lo que la audiencia necesita (ÚTIL) y lo que sirve al negocio
(RELEVANTE). CUR responde a dos preguntas:

1. **¿Qué contenido busca mi audiencia que yo puedo darle mejor que nadie?**
2. **¿Qué contenido sirve a mi negocio sin volverse un anuncio disfrazado?**

El contenido que solo es útil (pero no relevante) genera audiencia sin
negocio. El contenido que solo es relevante (pero no útil) es publirreportaje
que la gente ignora. El sweet spot está en la intersección.

## Las 3 Zonas de CUR

```
        ÚTIL para la audiencia
        ┌──────────────────────┐
        │                      │
        │   ZONA               │
        │   EDUCATIVA          │   ZONA SWEET SPOT
        │   (audiencia crece,  │   ┌──────────────────┐
        │    negocio no)       │   │ Contenido que la  │
        │                      │   │ audiencia AMA y   │
        │                      │   │ el negocio NECESITA│
        └──────────────────────┘   └──────────────────┘

        RELEVANTE para el negocio
        ┌──────────────────────┐
        │   ZONA               │
        │   PROMOCIONAL        │
        │   (negocio habla,    │
        │    audiencia huye)   │
        │                      │
        └──────────────────────┘
```

## Metodología — 4 pasos

### Paso 1: Inventario de necesidades (ÚTIL)

Desde los insights de Fase A (Buyer Personas) y Fase B (Alma de Marca),
lista todo lo que la audiencia necesita saber, resolver, o entender:

- ¿Qué preguntas hace tu audiencia antes de decidir?
- ¿Qué dolores tiene que tu creación resuelve?
- ¿Qué información busca y no encuentra fácilmente?
- ¿Qué mitos o confusiones existen en tu industria?

**Output:** Lista de 10+ necesidades reales de la audiencia.

### Paso 2: Inventario de objetivos (RELEVANTE)

Lista todo lo que el negocio necesita comunicar para existir y crecer:

- ¿Qué acciones quieres que tome la audiencia?
- ¿Qué categorías de creación necesitas mover?
- ¿Qué diferenciadores te separan de alternativas?
- ¿Qué objeciones necesitas disolver antes de la decisión?

**Output:** Lista de 8+ objetivos de negocio.

### Paso 3: Encontrar el sweet spot (CRUCE)

Cruza ambas listas. Para cada necesidad de la audiencia, pregúntate:
"¿Puedo resolver esto de una forma que también sirva a mi negocio?"

Los cruces que funcionan se convierten en **pilares de contenido**.

| Necesidad audiencia | Objetivo negocio | Pilar resultante |
|---------------------|------------------|------------------|
| "No sé si mi bebé va bien" | Vender consultas pediátricas | "Señales de desarrollo: cuándo sí y cuándo no preocuparse" |
| "Quiero invertir pero no sé cómo" | Vender asesoría financiera | "Primeros pasos para invertir sin miedo" |
| "Mi perro no me obedece" | Vender curso de adiestramiento | "Errores comunes que confunden a tu perro" |

**Gate:** Deben salir al menos 3 pilares. Si salen menos, vuelve al paso 1 — las necesidades no están suficientemente exploradas.

### Paso 4: Expandir cada pilar (MATRIZ CUR)

Para cada pilar, define:

| Elemento | Descripción |
|----------|-------------|
| **Pilar** | Nombre del pilar (1 frase) |
| **Necesidad que cubre** | Qué dolor, duda o curiosidad resuelve |
| **Objetivo de negocio** | Qué acción o percepción empuja |
| **Temas** | 3-5 temas concretos dentro del pilar |
| **Formatos sugeridos** | Stories, reels, carrusel, video, artículo |
| **Gancho emocional** | La emoción que activa el interés (curiosidad, alivio, urgencia, pertenencia) |

## Output: Matriz CUR

```markdown
## Matriz CUR — {Nombre del invitado}

### Pilar 1: {Nombre del pilar}
- **Necesidad:** {qué resuelve para la audiencia}
- **Objetivo:** {qué logra para el negocio}
- **Temas:**
  1. {tema}
  2. {tema}
  3. {tema}
- **Formatos:** {formatos}
- **Gancho:** {emoción}

### Pilar 2: {Nombre del pilar}
...
```

## Anti-patrones

| Error | Por qué falla | Corrección |
|-------|---------------|------------|
| "Consejos para tu negocio" | Demasiado genérico — no resuelve nada específico | Anclar a un dolor concreto de una buyer persona |
| "10 razones para elegirnos" | Es publirreportaje, no contenido útil | Convertir en "10 cosas que debes saber antes de elegir X" |
| Pilar único ("todo sobre salud") | No hay variedad de ángulos | Partir en subtemas: prevención, diagnóstico, tratamiento, estilo de vida |
| Solo educativo, cero negocio | La audiencia crece pero no convierte | Cada pilar debe tener un puente natural al siguiente paso |

## Gate de validación

Antes de dar por terminada la Fase C, verificar:

- [ ] Al menos 3 pilares definidos
- [ ] Cada pilar anclado a una necesidad real de Fase A
- [ ] Cada pilar conectado a un objetivo de negocio de Fase B
- [ ] Al menos 2 formatos sugeridos por pilar
- [ ] Ningún pilar es publirreportaje disfrazado

Si algún check falla, volver al paso correspondiente.

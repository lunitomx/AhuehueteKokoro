## Core Field Pattern

The repeated consulting pattern is not "make a campaign." It is:

1. Ground the business in purpose, objective, margin and readiness.
2. Clarify the invited person, the change event and the forces behind choice.
3. Convert the forces into hooks, stories, landing sections and follow-up.
4. Connect campaign learning to CRM, WhatsApp, tracking and lifetime value.
5. Use AI as an operator only after context, permissions and review gates exist.

## Router Implications

When a user asks for tactical execution, Kokoro should classify the request by
system bottleneck before naming a skill:

| Signal | First question | Route |
|--------|----------------|-------|
| Messy business context | What result must this move? | `/kokoro-diagnose` or `/kokoro-growth-diagnosis-run` |
| Campaign, hooks, content, landing | What forces and promise are already validated? | `/kokoro-campaign-lab-run` |
| Ads performance | What tracking and creative corpus exist? | `/kokoro-ads`, `/kokoro-gads`, `/kokoro-tracking-check` |
| WhatsApp, CRM, lost leads | Where does follow-up break? | `/kokoro-growth-diagnosis-run` |
| AI, copilots, agents | What process and permission can the AI operate? | `/kokoro-ai-copilot-run` |

## Required Gates For Runs

Every run created from this corpus must check:

- Purpose and annual/cycle objective.
- Invited person and non-invited person.
- Validated creation/model or the riskiest unvalidated assumption.
- Customer Forces or current substitute evidence.
- Measurement: primary metric, tracking, source of truth.
- Follow-up: CRM, WhatsApp, owner, cadence and next action.
- Privacy/export boundary.

## Candidate Runs

### `/kokoro-growth-diagnosis-run`

Use when the request is about growth but the bottleneck is unclear. It should
map soil, seed, demand, offer, tracking, follow-up and economics before
recommending tactical action.

### `/kokoro-campaign-lab-run`

Use when the user wants campaign assets or content. It should convert Customer
Forces into hooks, story, creative angles, landing sections, channel test and
follow-up plan.

### `/kokoro-ai-copilot-run`

Use when the user wants AI, agents or copilots. It should define the process to
delegate, allowed data, permissions, review gates, pilot task and stop rules.

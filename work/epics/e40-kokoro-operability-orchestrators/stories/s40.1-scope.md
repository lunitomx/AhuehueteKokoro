# S40.1 Scope: Public Repo Safety Gate

## Objective

Harden the public repository boundary so Kokoro can be shared without accidentally publishing guest data, local RaiSE runtime state, credentials, exports, reports, or generated private assets.

## In Scope

- Expand `.gitignore` for Kokoro runtime/private workspace artifacts.
- Strengthen the public privacy protocol for guests, platform identifiers, and MCP credentials.
- Add a share-readiness checklist that can be run before publishing or inviting collaborators.
- Verify no obvious real secrets are introduced by this story.
- Keep examples fake and generic.

## Out of Scope

- Building a pre-commit hook.
- Connecting real MCP servers.
- Migrating existing private guest folders.
- Auditing every historical commit.
- Creating the full orchestrator router.

## Done Criteria

- `.gitignore` covers private runtime and generated guest artifacts.
- Privacy protocol clearly separates public repo from private workspace.
- Share-readiness checklist exists and is linked from privacy protocol.
- Secret scan returns only placeholders or documentation examples.
- Epic progress marks `S40.1` as complete when story closes.

## Risk Notes

- `.raise/` can contain harmless project state today but sensitive local state later; ignore it broadly by default.
- Public examples must use `cliente_NN`, fake IDs, and placeholder tokens only.
- Runtime documentation must not imply MCP credentials belong in this repo.

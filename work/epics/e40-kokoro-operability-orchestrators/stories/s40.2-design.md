# S40.2 Design: Executive Router

## Complexity

Moderate.

The story changes a primary human entrypoint and adds routing knowledge. It does not add executable code, but it affects how a new business user experiences Kokoro.

## Problem

Kokoro has many useful skills, but a founder or marketing leader should not need to know internal command names to get value. The current `/kokoro` router focuses on strategic phases and does not yet route operational requests like "review Google Ads" or "prepare my marketing week."

## Value

A business user can state the outcome they want and Kokoro will identify the right path, distinguish strategy from operating work, and avoid overpromising planned orchestrators.

## Approach

- Extend `.claude/commands/kokoro.md` with an executive intent layer before the existing phase diagnosis.
- Add `.claude/knowledge/kokoro-executive-router.md` as the route map.
- Keep existing Fase 1-3 diagnosis intact for foundational strategy.
- Mark E40 orchestrators as planned until their command files exist.
- Provide "available now" fallback skills for planned orchestrators.

## Examples

### Example 1: Foundational Strategy

Input:

```text
No se por donde empezar con mi negocio.
```

Route:

```text
Primary route: /kokoro-diagnose
Reason: first clarify current terrain before recommending an operating workflow.
```

### Example 2: Google Ads Review

Input:

```text

Primary route: /kokoro-google-ads-run (planned)
Available now: /kokoro-gads + /kokoro-connect + /kokoro-analytics
First gates: guest resolved, Google Ads MCP connected, no credentials in repo.
```

## Acceptance Criteria

### MUST

- `/kokoro` distinguishes "I need strategic clarity" from "I need an operating decision."
- Route map covers at least six business intents from the story scope.
- Planned orchestrators are explicitly labeled planned.
- Every data-backed operating route names privacy and MCP gates.
- Existing phase-router guidance remains available.

### SHOULD

- The route map should be readable in under five minutes.
- Router language should stay direct and business-facing.

### MUST NOT

- Do not claim that planned E40 orchestrators are already implemented.
- Do not ask for tokens, credentials, or private guest data in public files.
- Do not remove the current Fase 1-3 methodology.

# S40.2 Scope: Executive Router

## Objective

Create a business-facing router that helps founders, marketing directors, and sales leaders choose the right Kokoro process from plain-language intent.

## In Scope

- Update `/kokoro` so it can route both strategic phase work and executive operating runs.
- Add a knowledge reference that maps business situations to Kokoro workflows.
- Label planned E40 orchestrators honestly until their command contracts exist.
- Preserve the existing Fase 1-3 routing behavior for foundational strategy.
- Include privacy and MCP caveats in operating-run routes.

## Out of Scope

- Implementing the actual E40 orchestrator commands.
- Connecting MCP servers.
- Rewriting every skill.
- Changing guest registry behavior.
- Creating a UI.

## Done Criteria

- `.claude/commands/kokoro.md` includes an executive-router layer.
- A supporting knowledge file exists for business-intent routing.
- At least these intents are covered:
  - "I do not know where to start"
  - "Review Google Ads"
  - "How is marketing this week?"
  - "Launch a creation"
  - "Improve acquisition/sales funnel"
  - "Share Kokoro safely"
- Planned orchestrators are marked as planned, not currently implemented.
- Router points to available current skills when an orchestrator is not yet implemented.

## Risk Notes

- The router must not become a long menu that overwhelms the business user.
- It must not imply MCP data exists before health and platform gates pass.
- It must not remove the original diagnostic flow, because foundational strategy remains valuable.

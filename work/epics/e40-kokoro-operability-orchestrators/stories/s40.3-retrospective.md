- Later stories should replace planned router rows with available routes only when command files actually ship.

## Heutagogical Checkpoint

1. **What did you learn?**  
   The contract is the operating-system layer. Without it, each orchestrator would become a bespoke prompt.

2. **What would you change about the process?**  
   Keep reusable policy in knowledge files and specific behavior in command files. That boundary stayed clean here.

3. **Are there improvements for the framework?**  
   Yes: every data-backed Kokoro run should report gate status before recommendation.

4. **What are you more capable of now?**  
   Future orchestrators can now be built from a shared standard instead of repeated judgment calls.

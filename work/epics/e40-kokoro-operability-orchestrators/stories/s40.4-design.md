# S40.4 Design: Google Ads Run

## Complexity

Complex.

The implementation is markdown contract work, but the recommendation risk is high. A weak Google Ads run could imply live performance without MCP evidence, recommend premature bidding changes, or ask for credentials in the wrong place.

## Problem

Business users ask simple questions such as "revisa Google Ads", but the correct process is not simple. Kokoro must resolve context, verify Google Ads MCP health, gather the right datasets, load Google Ads doctrine, interpret rhythm windows, and only then recommend action.

## Value

The run makes Kokoro usable by a founder, marketing director, or sales leader without requiring them to remember `/kokoro-connect`, `/kokoro-analytics`, `/kokoro-gads`, and tracking checks in the right order.

## Approach

Create `/kokoro-google-ads-run` as the executive operating contract. It will not replace `/kokoro-gads`; it will orchestrate it after context, MCP, dataset, knowledge, privacy, and permission gates pass.

## Components

| Component | Change | Purpose |
|---|---|---|
| `.claude/commands/kokoro-google-ads-run.md` | Create | Business-facing Google Ads orchestrator |
| `.claude/knowledge/kokoro-executive-router.md` | Modify | Mark Google Ads run as available and keep fallbacks |
| `work/epics/.../stories/s40.4-plan.md` | Create | Implementation tasks |
| `.raise/artifacts/s40.4-design.yaml` | Create | Typed design artifact |

## Domain Rules Preserved

- Google Ads is interpreted through signal, rhythm, and patience, not reactive "fixes".
- Missing MCP means no live account claim.
- Missing dataset means lower confidence, not guessing.
- Bidding strategy changes need maturity checks.
- Search Terms, Keywords, Auction Insights, conversions, landing context, and tracking context have different review rhythms.
- Action tools are never called without explicit invitation.

## Example: MCP Missing

Input:

```text
/kokoro-google-ads-run cliente_01 ultimos 30 dias
```

Expected output behavior:

```markdown
| Gate | Status | Evidence |
|---|---|---|
| Objective clear | Pass | Review Google Ads for cliente_01 |
| Context resolved | Partial | Placeholder context only |
| MCP registered | Blocked | google-ads MCP not callable in this runtime |
| MCP healthy | Blocked | No discovery call available |
| Data completeness | Blocked | No live datasets queried |
| Knowledge loaded | Pass | kokoro-mcp-google-ads.md, Service_Based_Checklist.md |
| Privacy check | Pass | no credentials or exports requested |

### Executive Read
No puedo afirmar rendimiento real de la cuenta todavia. El siguiente paso es conectar o verificar Google Ads MCP.
```

## Example: Search Campaign Run

Input:

```text
/kokoro-google-ads-run cliente_01 Search last_30_days
```

## Acceptance Criteria

### MUST

- Create `/kokoro-google-ads-run` with the shared orchestrator phases.
- Include a Google Ads dataset completeness table with Pass, Partial, Blocked, and Skipped handling.
- Include knowledge load rules for general account, review rhythm, Search, Display, and Performance Max.
- Include fallback behavior for missing MCP, unhealthy MCP, missing account mapping, unavailable dataset, and unclear date range.
- Include explicit permission boundaries for every action-capable Google Ads tool.
- Use only fake IDs and placeholders in examples.

### SHOULD

- Keep the command concise enough to be readable by a business user.
- Point unsupported setup work to `/kokoro-connect` and `kokoro-mcp-google-ads.md`.
- Include a short demo path for non-connected accounts.

### MUST NOT

- Ask for API keys, access tokens, refresh tokens, or client secrets in chat.
- Claim performance when MCP gates are blocked.
- Commit exports, reports, screenshots, account IDs, or guest-specific artifacts.
- Execute mutations such as budgets, bidding, keywords, negatives, ads, targeting, or campaign status without explicit invitation.

# S40.4 Scope: Google Ads Run

## In Scope

- Create the `/kokoro-google-ads-run` command contract.
- Chain existing Kokoro capabilities instead of replacing them:
  - `/kokoro-connect`
  - `/kokoro-analytics`
  - `/kokoro-gads`
  - `/kokoro-tracking-check`
- Apply the shared `kokoro-orchestrator-contract.md`.
- Define Google Ads-specific gates for:
  - invitado or temporary context
  - Google Ads MCP registration and health
  - account selection
  - date range
  - campaign type
  - dataset completeness
  - knowledge loaded
  - privacy
  - recommendation only
  - explicit permission before action
- Define the required dataset checklist for Search, Display, and Performance Max diagnostics.
- Define fallback behavior when MCP is missing, unhealthy, or incomplete.
- Keep every example public-safe with fake IDs.

## Out of Scope

- Registering real MCP servers.
- Connecting real Google Ads accounts.
- Reading or saving private guest exports.
- Creating dashboards or reports.
- Mutating real campaigns.
- Implementing an executable MCP wrapper.
- Rewriting `/kokoro-gads`.

## Done Criteria

- `.claude/commands/kokoro-google-ads-run.md` exists and is business-facing.
- The command names every knowledge file it may load before recommendation.
- The command includes a required dataset table with pass/partial/blocked behavior.
- The command explicitly routes missing MCP setup to `/kokoro-connect` or MCP reference docs.
- The command includes action boundaries for budgets, bidding, keywords, negatives, ads, campaign status, targeting, and landing changes.
- The executive router marks `/kokoro-google-ads-run` as available instead of planned.
- E40 progress tracking records S40.4 completion only after review and merge.

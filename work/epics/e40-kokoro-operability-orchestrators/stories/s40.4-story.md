# S40.4 Story: Google Ads Run

## User Story

As a founder, marketing director, or sales leader,
I want a single Kokoro Google Ads operating run,
so that I can ask for a campaign review and receive a grounded diagnosis without needing to know which tactical skills, MCP checks, datasets, and knowledge files to run first.

## Context

The existing `/kokoro-gads` skill contains strong Google Ads methodology, but it behaves as a tactical optimization guide. E40 needs a business-facing orchestrator that starts with context and connection health, gathers the right datasets, loads the right knowledge, and only then recommends what to wait on, inspect, or change.

## Acceptance Criteria

### AC1: MCP and Context Gates Before Analysis

Given a user asks Kokoro to review Google Ads,
when `/kokoro-google-ads-run` is invoked,
then the run must first resolve the invitado or temporary context,
and it must verify Google Ads MCP registration/health before making any live platform claim.

### AC2: Required Dataset Checklist

Given the Google Ads MCP is available,
when the run performs a diagnostic,
then it must define and report the status of required datasets:
campaigns, performance, status and budget, keywords, search terms, negatives or exclusions where available, bidding, conversions, demographics, geography, auction or competition context where available, and landing/tracking context.

### AC3: Knowledge Before Recommendation

Given the run knows the campaign type,
when it prepares recommendations,
then it must load the general Google Ads knowledge, the service rhythm checklist, and the campaign-type-specific knowledge before making any optimization recommendation.

### AC4: Patience and Permission Gates

Given the run finds an apparent opportunity,
when the evidence is incomplete or the optimization window has not matured,
then it must recommend waiting or verifying before action,
and it must ask explicit permission before any account mutation such as budget, bidding, keywords, negatives, ads, or campaign status changes.

### AC5: Safe Public Artifact

Given this is a public repository,
when the story is complete,
then the command contract and documentation must use fake IDs and placeholders only,
and no private account data, exports, credentials, or guest-specific reports may be committed.

## Scenario Examples

### Scenario: MCP Missing

Input:

```text
/kokoro-google-ads-run cliente_01 ultimos 30 dias
```

Expected behavior:

- Context gate can pass with `cliente_01` placeholder.
- MCP registered/healthy gates are blocked.
- Kokoro offers `/kokoro-connect` or MCP setup.
- Kokoro does not claim performance, CPC, CPA, search-term, or conversion facts.

### Scenario: MCP Healthy With Search Campaigns

Input:

```text
/kokoro-google-ads-run cliente_01 Search last_30_days
```

Expected behavior:

- Kokoro lists datasets queried.
- Search Terms and Keywords are required.
- `Optimisaciones_Google_Ads.md`, `Service_Based_Checklist.md`, `Optimisa_Tus_Campanas_Busqueda.md`, and `Keyword_Targeting_Search_Term_Audits_Bidding_DETAILED.md` are loaded before recommendations.
- Output separates findings, assumptions, what not to touch yet, recommended next actions, and permission gate.

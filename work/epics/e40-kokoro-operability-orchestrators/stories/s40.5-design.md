# S40.5 Design: Weekly Marketing Pulse Run

## Complexity

Complex.

The work is mostly command-contract documentation, but the interpretation risk is high. A weekly pulse can easily become a dashboard dump, a platform ranking, or a set of premature actions unless it explicitly gates data coverage, trend comparison, tracking confidence, and permission boundaries.

## Problem

Business leaders do not want to manually run `/kokoro-scorecard`, `/kokoro-analytics`, Google Ads checks, and tracking checks every week. They want one operating rhythm: what changed, what it means, what needs attention, and what should wait.

## Value

Kokoro becomes useful as a weekly marketing operating layer, not just a collection of metrics skills. The user can ask "como va marketing esta semana?" and receive a decision-ready read.

## Approach

Create `/kokoro-weekly-marketing-run` as the executive weekly pulse. It orchestrates `/kokoro-connect`, `/kokoro-analytics`, `/kokoro-scorecard`, `/kokoro-google-ads-run`, and `/kokoro-tracking-check` without replacing them.

## Domain Rules Preserved

- Weekly pulse is not a dashboard dump.
- Missing platform data is visible, not silently omitted.
- Trends matter more than isolated values.
- Cross-platform interpretation is the value layer.
- Tracking confidence determines how strongly the pulse can speak.
- Avoid binary ad framing such as winner/loser or best/worst.
- Reports and exports with real platform data are private artifacts.

## Example: Weekly Run With Partial Platforms

Input:

```text
/kokoro-weekly-marketing-run cliente_01 last_7_days
```

Expected behavior:

```markdown
| Gate | Status | Evidence |
|---|---|---|
| Context resolved | Pass | cliente_01 placeholder |
| Date range | Pass | last_7_days vs previous_7_days |
| Platform connections | Partial | Meta Ads and GA4 connected; Google Ads and GSC skipped |
| Data completeness | Partial | paid + site data only |
| Privacy check | Pass | no report persisted publicly |

### Executive Read
La senal pagada crecio, pero la continuidad hacia sitio esta parcial porque solo hay GA4 y Meta conectados.
```

## Example: No MCP Available

Input:

```text
/kokoro-weekly-marketing-run cliente_01 esta semana
```

Expected behavior:

- MCP gates are blocked.
- Kokoro does not invent metrics.
- Kokoro offers `/kokoro-connect`.
- Kokoro may provide a weekly pulse template without live data.

## Acceptance Criteria

### MUST

- Create `/kokoro-weekly-marketing-run`.
- Include gates for objective, context, date range, platform connections, MCP health, data completeness, knowledge, privacy, recommendation-only, and action invitation.
- Define platform-specific datasets for Meta Ads, Google Ads, GA4, and Search Console.
- Compare the selected weekly period with the previous equivalent period.
- Include cross-platform interpretation, not just metrics.
- Include "What I Would Not Touch Yet".
- Avoid best/worst, winner/loser campaign language.
- Use public-safe placeholders only.

### SHOULD

- Keep scorecard detail below the executive read.
- Point unconnected platforms to `/kokoro-connect`.
- Point tracking uncertainty to `/kokoro-tracking-check`.

### MUST NOT

- Commit private weekly reports or exports.
- Ask for tokens or credentials in chat.
- Mutate campaigns, budgets, content, tracking, or site settings without explicit invitation.
- Treat a missing MCP as zero performance.

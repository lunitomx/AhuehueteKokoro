# S40.5 Scope: Weekly Marketing Pulse Run

## In Scope

- Create `.claude/commands/kokoro-weekly-marketing-run.md`.
- Orchestrate existing capabilities:
  - `/kokoro-connect`
  - `/kokoro-analytics`
  - `/kokoro-scorecard`
  - `/kokoro-google-ads-run`
  - `/kokoro-tracking-check`
- Apply `kokoro-orchestrator-contract.md`.
- Define weekly date-range behavior and comparison period.
- Define connected-platform gates for:
  - Meta Ads
  - Google Ads
  - GA4
  - Google Search Console
- Define cross-platform interpretation:
  - traffic quality
  - signal distribution
  - paid-to-site continuity
  - search demand
  - conversion/tracking confidence
  - what not to touch yet
- Avoid binary ranking language such as winner/loser or best/worst campaign framing.
- Update executive router to mark the route available.

## Out of Scope

- Connecting real platform accounts.
- Running live MCP queries in this story.
- Creating dashboards, spreadsheets, or PDFs.
- Saving real guest weekly reports in the public repo.
- Mutating platform settings.
- Rewriting `/kokoro-scorecard`.

## Done Criteria

- `/kokoro-weekly-marketing-run` exists as a command contract.
- The command includes gates for context, date range, platform connection, MCP health, data completeness, knowledge loaded, privacy, and permission.
- The command defines platform-specific datasets and graceful degradation.
- The command output template includes an executive read, platform coverage, cross-platform interpretation, what not to touch yet, recommended next actions, and permission gate.
- Router marks `/kokoro-weekly-marketing-run` as available.
- Verification confirms no credentials, real account IDs, or private report examples were added.

# S40.5 Story: Weekly Marketing Pulse Run

## User Story

As a founder, marketing director, or sales leader,
I want Kokoro to run a weekly marketing pulse across connected platforms,
so that I can understand what changed, what needs attention, and what should not be touched yet without manually invoking separate analytics, scorecard, and platform skills.

## Context

Kokoro already has `/kokoro-scorecard`, `/kokoro-analytics`, and platform MCP references. Those pieces need an executive orchestrator that checks context, platform connections, date range, data completeness, knowledge files, privacy, and permission boundaries before producing a weekly operating read.

## Acceptance Criteria

### AC1: Weekly Pulse Route

Given a user asks how marketing performed this week,
when `/kokoro-weekly-marketing-run` is invoked,
then Kokoro must resolve the invitado, select a weekly date range, inspect connected platforms, and report unavailable platforms without failing the whole run.

### AC2: Cross-Platform Dataset Coverage

Given platforms are connected,
when the run gathers data,
then it must define dataset coverage for Meta Ads, Google Ads, GA4, and Search Console where available,
and it must mark each platform as Pass, Partial, Blocked, or Skipped.

### AC3: Executive Interpretation

Given platform data or partial data is available,
when Kokoro presents the pulse,
then it must translate metrics into business direction: trend, signal distribution, risk, opportunity, what not to touch yet, and next action.

### AC4: Privacy and Permission Boundaries

Given weekly reports may contain sensitive platform data,
when the run creates output,
then it must avoid committing guest reports, exports, IDs, screenshots, or private findings to the public repo,
and it must ask explicit permission before saving private reports or mutating platform/account settings.

### AC5: Router Availability

Given the command contract is complete,
when the router is updated,
then `/kokoro-weekly-marketing-run` must be marked as available.

## Scenario Examples

### Scenario: One Platform Connected

Input:

```text
/kokoro-weekly-marketing-run cliente_01 esta semana
```

Expected behavior:

- Context gate passes with placeholder.
- Connected platform is queried.
- Unconnected platforms are shown as Skipped with setup next step.
- Executive read is based only on available data.

### Scenario: No MCP Available

Input:

```text
/kokoro-weekly-marketing-run cliente_01 last_7_days
```

Expected behavior:

- MCP gates are blocked.
- Kokoro does not invent metrics.
- Kokoro gives setup path through `/kokoro-connect`.
- Kokoro can provide a methodology-only weekly pulse template if useful.

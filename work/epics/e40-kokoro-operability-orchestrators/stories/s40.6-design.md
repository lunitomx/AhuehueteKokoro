# S40.6 Design: Creative Campaign Run

## Complexity

Complex.

The implementation is a command contract, but the workflow is sensitive: if Kokoro jumps straight to image generation, it repeats the exact failure mode this story exists to prevent.

## Problem

Users often ask for "5 images", "copies", or "headlines" when what they really need is a campaign decision journey. Existing skills generate or review assets, but they do not force true promise, storyboard, visual attention, and review before production.

## Value

Kokoro becomes a creative director and conversion strategist, not just an image prompt generator. A carousel becomes a sequence that moves attention and decision.

## Approach

Create `/kokoro-creative-campaign-run` as the business-facing run. It chains `/kokoro-ads`, `/kokoro-creative`, and `/kokoro-creative-review`, using creative gates from `kokoro-orchestrator-contract.md`.

## Components

| Component | Change | Purpose |
|---|---|---|
| `.claude/commands/kokoro-creative-campaign-run.md` | Create | Creative campaign orchestrator |
| `.claude/knowledge/kokoro-executive-router.md` | Modify | Mark route available |
| `.raise/artifacts/s40.6-design.yaml` | Create | Typed design artifact |
| `work/epics/.../stories/s40.6-plan.md` | Create | Implementation plan |

## Domain Rules Preserved

- A creative asset must move a decision, not merely look polished.
- The promise must match the real event/offer.
- One frame equals one thought, one emotion, one advance.
- Visual direction must include gaze, focal point, hierarchy, CTA path, and brand non-negotiables.
- Review comes before Meta copy/targeting and before publishing.
- Asset generation and persistence require explicit permission.

## Example: Carousel Run

Input:

```text
/kokoro-creative-campaign-run cliente_01 carrusel Meta Ads sesion informativa
```

Expected flow:

1. Clarify the real promise and event reality.
2. Identify audience tension and desired decision.
3. Draft 5-frame story arc.
4. Define visual direction per frame.
5. Ask for storyboard/visual approval.
6. Build generation specs.
7. Review creative logic.
8. Draft Meta copy/targeting if requested.
9. Ask permission before generating, saving, or publishing.

## Frame Model

| Frame | Role | Strategic Question |
|---:|---|---|
| 1 | Hook | What stops the scroll and names the tension? |
| 2 | Tension | What doubt does the audience already feel? |
| 3 | Reframe | What changes how they understand the situation? |
| 4 | Choice or proof | What makes the next step feel grounded? |
| 5 | Invitation | What clear invitation closes the journey? |

## Acceptance Criteria

### MUST

- Create `/kokoro-creative-campaign-run`.
- Include `GATE-PROMISE-TRUE`, `GATE-STORYBOARD-APPROVED`, `GATE-VISUAL-DIRECTION-CLEAR`, and `GATE-CREATIVE-REVIEWED`.
- Include frame-by-frame carousel model.
- Include visual attention requirements by frame.
- Include generation and persistence permission gates.
- Reference `/kokoro-ads`, `/kokoro-creative`, `/kokoro-creative-review`, and required knowledge files.
- Use only public-safe placeholders.

### SHOULD

- Include a short "do not generate yet" fallback for missing strategy/storyboard.
- Include a compact output template that a business user can approve.
- Support non-carousel assets by treating them as single-frame campaigns.

### MUST NOT

- Generate images before strategy/storyboard approval.
- Save generated specs or assets in public repo paths.
- Ask for API keys in chat.
- Change brand/model/source assets unless the user explicitly invites that change.

# S40.6 Scope: Creative Campaign Run

## In Scope

- Create `.claude/commands/kokoro-creative-campaign-run.md`.
- Orchestrate existing capabilities:
  - `/kokoro-ads`
  - `/kokoro-creative`
  - `/kokoro-creative-review`
  - `/kokoro-analytics` when post-pauta data is requested
- Apply creative gates from `kokoro-orchestrator-contract.md`:
  - `GATE-PROMISE-TRUE`
  - `GATE-STORYBOARD-APPROVED`
  - `GATE-VISUAL-DIRECTION-CLEAR`
  - `GATE-CREATIVE-REVIEWED`
- Define a carousel story arc.
- Define visual direction requirements by frame.
- Define permission boundaries for generation, saving, exporting, publishing, or committing guest-specific assets.
- Update executive router to mark `/kokoro-creative-campaign-run` available.

## Out of Scope

- Generating real images in this story.
- Calling Gemini or any image API.
- Saving private campaign specs or assets.
- Rewriting `/kokoro-creative`, `/kokoro-creative-review`, or `/kokoro-ads`.
- Building a UI for carousel planning.

## Done Criteria

- `/kokoro-creative-campaign-run` exists as a command contract.
- The command includes strategy, promise, audience, storyboard, visual direction, generation spec, review, Meta copy, and permission phases.
- The command includes a carousel-specific frame model.
- The command prevents direct generation when promise/storyboard/visual direction gates are missing.
- Router marks `/kokoro-creative-campaign-run` as available.
- Verification confirms no private guest data, asset paths, or credentials were added.

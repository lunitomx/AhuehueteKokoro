# S40.6 Story: Creative Campaign Run

## User Story

As a founder, marketing director, or creative lead,
I want Kokoro to orchestrate visual campaign creation from strategy to storyboard to generation review,
so that carousels and paid creatives move a clear decision instead of becoming isolated attractive assets.

## Context

The existing `/kokoro-creative`, `/kokoro-creative-review`, and `/kokoro-ads` skills are useful tactical pieces. They do not yet force the full creative campaign sequence: true promise, audience tension, story arc, visual attention direction, generation specs, review, Meta copy, and permission before saving or publishing assets.

## Acceptance Criteria

### AC1: Strategy Before Production

Given a user asks for a carousel, ad visual, or campaign creative,
when `/kokoro-creative-campaign-run` is invoked,
then Kokoro must first clarify objective, true promise, audience, offer/event reality, and brand non-negotiables before copy or image generation.

### AC2: Storyboard Gate

Given the user wants a carousel or multi-frame asset,
when Kokoro prepares the campaign,
then it must define a frame-by-frame story arc before generating assets:
hook, tension, reframe, choice/proof, invitation.

### AC3: Visual Direction Gate

Given the story arc is defined,
when Kokoro prepares generation specs,
then each frame must include visual attention direction: gaze, focal point, hierarchy, CTA path, what to preserve, and what not to change.

### AC4: Review Before Publishing

Given a draft creative or generation spec exists,
when Kokoro evaluates it,
then it must run a creative review lens before Meta copy/targeting or any publish/save action.

### AC5: Public Safety

Given generated assets and campaign context may be private,
when the story is complete,
then the command must keep examples public-safe and require explicit permission before saving generated assets, reports, specs, or guest-specific files.

## Scenario Examples

### Scenario: Carousel Request

Input:

```text
/kokoro-creative-campaign-run cliente_01 carrusel Meta Ads sesion informativa
```

Expected behavior:

- Kokoro identifies the real promise and event reality.
- Kokoro produces storyboard before image prompts.
- Kokoro asks for approval before generation.
- Kokoro does not persist private assets in the public repo.

### Scenario: User Asks Directly For Images

Input:

```text
Hazme las 5 imagenes del carrusel.
```

Expected behavior:

- Kokoro does not skip strategy/storyboard.
- Kokoro asks for missing brand/source assets or uses already provided context.
- Kokoro defines frame roles and visual direction first.
- Kokoro asks permission before generating or saving assets.

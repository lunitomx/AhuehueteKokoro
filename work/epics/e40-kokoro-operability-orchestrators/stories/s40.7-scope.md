# S40.7 Scope: Launch Run

## In Scope

- Create `.claude/commands/kokoro-launch-run.md`.
- Orchestrate existing capabilities:
  - `/kokoro-canvas`
  - `/kokoro-forces`
  - `/kokoro-pescar`
  - `/kokoro-experiment`
  - `/kokoro-launch`
  - `/kokoro-landing`
  - `/kokoro-tracking-check`
  - `/kokoro-weekly-marketing-run`
- Apply `kokoro-orchestrator-contract.md`.
- Define launch readiness gates:
  - creation defined
  - invited audience clear
  - promise true
  - offer/event reality clear
  - validation evidence present or explicitly missing
  - primary metric and thresholds defined
  - tracking plan
  - asset plan
- Define fallback behavior for missing validation, missing landing, missing tracking, or unclear CTA.
- Update executive router to mark `/kokoro-launch-run` available.

## Out of Scope

- Creating real campaign assets.
- Publishing content.
- Editing websites or landing pages.
- Creating paid campaigns.
- Connecting real platform accounts.
- Saving private launch reports in the public repo.
- Rewriting the underlying tactical skills.

## Done Criteria

- `/kokoro-launch-run` exists as a command contract.
- The command includes a launch readiness table and method chain.
- The command includes output sections for evidence, assumptions, launch plan, asset plan, tracking, what not to touch yet, next actions, and permission gate.
- Router marks `/kokoro-launch-run` as available.
- Verification confirms no credentials, private paths, real guest data, or publish-action assumptions were added.

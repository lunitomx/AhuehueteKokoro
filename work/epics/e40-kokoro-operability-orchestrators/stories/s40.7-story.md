# S40.7 Story: Launch Run

## User Story

As a founder, marketing director, or launch owner,
I want Kokoro to orchestrate a launch from model clarity to launch assets and measurement,
so that I do not jump into copy, ads, or landing work before the creation, invited audience, validation, and tracking are ready.

## Context

Kokoro has strong tactical skills for canvas, forces, PESCAR, experiments, launch copy, landing evaluation, tracking, and scorecards. S40.7 creates the operating run that sequences them into a launch workflow for a business leader.

## Acceptance Criteria

### AC1: Launch Readiness Before Assets

Given a user asks to launch a creation,
when `/kokoro-launch-run` is invoked,
then Kokoro must first resolve context and verify launch readiness: creation, invited audience, promise, offer/event reality, validation status, and target metric.

### AC2: Method Chain

Given launch readiness is partial or complete,
when Kokoro builds the launch plan,
then it must chain relevant methods in order: canvas, forces, PESCAR, experiment, launch, landing, tracking, and scorecard.

### AC3: Evidence and Gaps

Given evidence is missing,
when Kokoro recommends next actions,
then it must separate validated facts, assumptions, blockers, and what should not be touched yet.

### AC4: Measurement and Tracking

Given the launch will create demand,
when Kokoro prepares the plan,
then it must include tracking readiness, primary metric, success threshold, invalidation threshold, and weekly readback path.

### AC5: Permission Boundaries

Given launch work may generate or publish assets,
when the run reaches implementation,
then Kokoro must ask explicit permission before saving private artifacts, publishing content, editing landings, creating campaigns, or changing tracking.

## Scenario Examples

### Scenario: Launch Request With No Validation

Input:

```text
/kokoro-launch-run cliente_01 nueva creacion
```

Expected behavior:

- Kokoro identifies validation as partial or blocked.
- Kokoro recommends canvas/forces/experiment before launch assets.
- Kokoro does not generate campaign assets as if the offer were validated.

### Scenario: Launch Request With Validation

Input:

```text
/kokoro-launch-run cliente_01 curso validado lanzamiento septiembre
```

Expected behavior:

- Kokoro checks launch readiness.
- Kokoro builds launch sequence and asset plan.
- Kokoro defines tracking and scorecard readback.
- Kokoro asks permission before creating or saving assets.

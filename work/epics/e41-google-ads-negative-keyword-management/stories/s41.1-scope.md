# S41.1 Scope: Contract Discovery for Negative Keywords

## Objective

Confirm the current Google Ads negative keyword connector contract and constraints to design `S41.2` safely.

## In Scope

- Map file locations and modules that currently handle Google Ads keyword actions.
- Capture exact input/output schema for existing negative keyword add operations.
- Verify whether `delete` is already available or if delete needs new action path.
- Confirm whether `state` tracking is persisted (`active`/`removed` or similar).
- Document compatibility implications for existing `add` workflows.

## Out of Scope

- Implementar cambios funcionales de `delete`.
- Cambiar la integración de MCP/SDK.
- Cambiar documentación de usuario final (se hace en S41.4).
- Soporte de lotes masivos >100 elementos (se evaluará después si aplica).

## Done Criteria

- [x] Mapeo de módulos/handlers documentado en este mismo archivo.
- [x] Schema actual de operación registrada con fields y constraints.
- [x] Riesgo de compatibilidad identificado y decisión de fallback definida.
- [x] `s41.2` no inicia sin estas evidencias.

## Verification Command Plan

Ejecutar en este orden durante el trabajo:

- `rg -n "add_negative|negative|keyword|google ads|google-ads|connector|mcp" <repo_or_paths>`
- `rg -n "add|delete|remove|state|status|match_type" <path_del_connector>`
- Revisar documentación MCP y configuración de herramientas de ejecución.

### Discovery Result (S41.1 Task 1)

- **Path in current checkout:** no se encontró módulo de conector Google Ads en código fuente público para esta revisión.
- **External source confirmed:** `googleads/google-ads-mcp` es la repo oficial revisada para el surface MCP.
- **Evidence from official repo:** `ads_mcp/tools/core.py`, `ads_mcp/tools/search.py`, `ads_mcp/tools/get_resource_metadata.py`, and `ads_mcp/tools_config.yaml`.
- **Observed surface:** `customers`, `search`, `metadata` only. No `add_negative_keywords`, no `delete`, no `remove`, no mutate tool for keywords.
- **Riesgo:** `code-path-gap` para `S41.2` si se intentara extender el proveedor oficial.
- **Decisión:** el contrato de `delete` debe resolverse como adaptación local de Kokoro, no como extensión del servidor oficial.

- Identificar repositorio o paquete de integración efectivo y registrar esa deuda como `code-path-gap` en este artifact para cerrar en planning del próximo sprint.

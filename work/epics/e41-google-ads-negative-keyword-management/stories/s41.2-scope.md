# S41.2 Scope: Action Contract for Delete

## Objective

Formalizar el contrato funcional de `negative_keyword.delete` como adaptación local de Kokoro, ya que el servidor oficial de Google Ads MCP no expone mutaciones de keywords.

## In Scope

- Definir payload de entrada y salida para eliminación de negativas.
- Definir estrategia de compatibilidad con flujos `add` actuales.
- Definir contrato de idempotencia y tratamiento de escenarios parciales (`not_found`, `already_removed`).
- Definir reglas de observabilidad (`audit`), incluyendo campos mínimos.
- Definir cómo la adaptación local delega a la surface oficial de solo lectura para descubrir contexto antes de operar.

## Out of Scope

- Implementar llamadas reales de conector en runtime.
- Cambiar provider/SDK de Google Ads.
- Soporte de lotes grandes (`>100`) fuera del diseño base.
- Cambiar documentación de uso final (S41.4).

## Dependencies

- S41.1 debe cerrar con `connector_path` y evidencia verificable.
- Issue #1 define el alcance de negocio y criterio de aceptación final.
- La fuente oficial revisada es `googleads/google-ads-mcp`; no ofrece delete nativo.

## Done Criteria

- [x] Request/response contract aprobado para `negative_keyword.delete`.
- [x] `compatibility` y rollback/fallback definidos (`add` sin ruptura).
- [x] Riesgo técnico documentado (`soft-delete` vs `hard-delete`).
- [x] Plan de implementación de S41.3 puede arrancar sin ambigüedad.

## Verification

- Validar que todos los campos del contrato estén en inglés/esquema consistente con el adapter local.
- Validar que los campos de error sean explícitos y por elemento.
- Validar que la fuente oficial solo se use para discovery/lookup, no para mutación.

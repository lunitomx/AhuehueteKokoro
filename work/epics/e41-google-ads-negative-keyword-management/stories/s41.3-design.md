# S41.3 Design: Local Delete Adapter

## Complexity

Moderate to high, because the provider is read-only and the mutation must be modeled locally.

## Problem

The official Google Ads MCP server does not expose negative keyword mutation tools. A local adapter is required if Kokoro wants to support `negative_keyword.delete` without forking the provider.

## Approach

1. Use official MCP tools for discovery:
   - `list_accessible_customers`
   - `search`
   - `get_resource_metadata`
2. Resolve the target negative keyword locally.
3. Emit a local delete operation with audit/event parity.
4. Return partial/failure states consistently with `add`.

## Adapter Contract

- Input: `action`, `text`, `match_type`, `scope`, optional `dry_run`.
- Output: `status`, `removed`, `not_found`, `already_removed`, `errors`, `audit_event_id`.
- Behavior: idempotent, backward compatible, no provider mutation assumption.

## Implementation Guardrail

If the runtime has no local execution path for this adapter, do not fake success. Keep the story blocked until the code surface exists.

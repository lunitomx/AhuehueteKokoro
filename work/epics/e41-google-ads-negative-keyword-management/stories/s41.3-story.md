# S41.3 Story: Implement Local Delete Adapter

## User Story

Como operador de Kokoro,
quiero una operación local para eliminar keywords negativas,
para ejecutar limpieza trazable aunque el servidor oficial de Google Ads MCP sea solo de lectura.

## Acceptance Criteria

### AC1: Adapter local definido

Debe existir una capa local de Kokoro que exponga `negative_keyword.delete` y resuelva la intención usando el contrato formalizado en S41.2.

### AC2: Discovery previo a mutación

La operación debe usar el servidor oficial de Google Ads MCP solo para discovery/lookup (`search`, `metadata`, `list_accessible_customers`) antes de cualquier mutación local o downstream.

### AC3: Contract parity

La respuesta debe conservar paridad con `add` en:

- trazabilidad
- IDs/criterios afectados
- errores parciales
- evento de auditoría

### AC4: Safety

La ejecución debe ser idempotente y no debe romper los flujos existentes de `add`.

### AC5: Implementability check

No se inicia implementación hasta que exista un path de runtime local confirmado en este checkout o en el paquete de ejecución correspondiente.

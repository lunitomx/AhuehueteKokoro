# S41.4 Design: Audit + Docs + Regression Coverage

## Complexity

Low to moderate. The main challenge is keeping examples public-safe while reflecting the local adapter contract.

## Problem

Even with the contract defined, the workflow still needs durable artifacts:

- audit parity
- user-facing examples
- regression checklist

## Approach

1. Document the audit envelope used by `negative_keyword.delete`.
2. Provide safe examples with placeholders only.
3. Define regression cases at the contract layer.
4. Keep `add` explicitly untouched.

## Deliverables

- Updated epic/story docs
- Example invocation snippets
- Regression checklist
- Audit field contract

## Guardrails

- No credentials or real customer IDs.
- No claim that the official Google Ads MCP supports mutation.
- No regression story that implies `add` changed behavior.

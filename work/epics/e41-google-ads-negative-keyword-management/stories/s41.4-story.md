# S41.4 Story: Audit + Docs + Regression Coverage

## User Story

Como operador de Kokoro,
quiero evidencia de auditoría, documentación y cobertura de regresión para eliminación de negativas,
para poder operar la limpieza sin romper la trazabilidad ni la experiencia actual de `add`.

## Acceptance Criteria

### AC1: Audit parity

La operación local debe dejar registro equivalente a `add`:

- `connector`
- `action`
- `payload_hash`
- `outcome`
- `actor`
- `timestamp`

### AC2: Documentation

Debe existir documentación de uso para la eliminación de negativas con ejemplos públicos y sin credenciales reales.

### AC3: Regression coverage

Debe existir cobertura de regresión para:

- `not_found`
- `already_removed`
- `partial_success`
- `add` sin regresión

### AC4: Safe closure

La historia solo se cierra si hay path de runtime o, en su defecto, una declaración explícita de bloqueo con siguiente acción concreta.

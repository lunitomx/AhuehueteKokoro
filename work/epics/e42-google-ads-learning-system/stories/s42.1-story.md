# S42.1 Story: Learning Record Schema

## Story
Define the learning record schema that Kokoro will use to capture plan,
execution, and learning for Google Ads sessions without breaking the current
`session_log` format.

## Outcome
The repo has a shared record model that can represent a Google Ads review,
the changes made, the learning captured, and the next action.

## Acceptance
- The schema is additive to `session_log`.
- Existing entries remain valid.
- Google Ads-specific fields are optional.
- The schema does not introduce runtime secrets or MCP wiring.

## Dependencies
- `.claude/knowledge/kokoro-session-log.md`
- `.claude/knowledge/kokoro-learning-dashboard.md`

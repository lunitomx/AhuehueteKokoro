# S45.1 Story: Meta Ads Learning State Detector

## User Story

Como operador de Kokoro,
quiero que el learning_state de las campañas de Meta Ads se infiera
automáticamente desde los datos del MCP,
para no depender de mi memoria al cerrar cada sesión.

## Acceptance Criteria

### AC1: Detector implementado
Existe un contrato documentado que define cómo inferir learning_state
desde los datos del MCP de Meta Ads (spend, conversions, CPA trend, campaign age).

### AC2: Thresholds claros
Los thresholds de detección están documentados y son verificables:
- <7 días activa → learning
- <50 conversiones en 7 días → learning
- >50 conversiones + CPA estable (±15%) → stable
- CPA subió >30% → needs_attention
- 0 conversiones con spend >0 en 7 días → needs_attention

### AC3: Fallback sin MCP
Si el MCP no está disponible, learning_state se omite (no se inventa).

### AC4: Override manual
El operador puede sobrescribir el learning_state inferido si discrepa.

### AC5: Integración con session_log
El learning_state inferido se persiste en el session_log al cerrar sesión.
